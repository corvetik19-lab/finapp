# Инструкция по добавлению тестовых тендеров

## Описание

Скрипт `seed-tenders-additional.sql` добавляет 3 тендера с разными типами закупки для отображения в карточках.

## Добавляемые тендеры

### 1. Поставка компьютерной техники (ФЗ-44)
- **Номер закупки:** 0373200000123000010
- **Заказчик:** ГБОУ "Средняя общеобразовательная школа №15"
- **НМЦК:** 4 500 000 ₽
- **Город:** Санкт-Петербург
- **Этап:** Новые контракты в реализацию
- **Площадка:** ЭТП ГПБ

### 2. Услуги по охране объекта (ФЗ-223)
- **Номер закупки:** 0373300000123000020
- **Заказчик:** ОАО "Российские железные дороги"
- **НМЦК:** 7 200 000 ₽
- **Город:** Москва
- **Этап:** Анализ и просчёт
- **Площадка:** Сбербанк-АСТ

### 3. Поставка инновационного оборудования (ФЗ-275)
- **Номер закупки:** 0373400000123000030
- **Заказчик:** ФГБУ "Национальный исследовательский центр"

## Как применить

### Вариант 1: Через Supabase Dashboard (SQL Editor)

1. Откройте Supabase Dashboard
2. Перейдите в **SQL Editor**
3. Создайте новый запрос
4. Скопируйте содержимое файла `seed-tenders-additional.sql`
5. Вставьте в редактор
6. Нажмите **Run** или `Ctrl+Enter`

### Вариант 2: Через Supabase CLI

```bash
# Находясь в корне проекта
supabase db execute --file db/seed-tenders-additional.sql
```

### Вариант 3: Через psql

```bash
psql -h your-project.supabase.co -U postgres -d postgres -f db/seed-tenders-additional.sql
```

## Проверка

После выполнения скрипта:

1. Откройте приложение
2. Перейдите в **Тендеры** → **Реестр тендеров**
3. Переключитесь на вид **Карточки**
4. Вы должны увидеть 3 новых тендера с типами:
   - ФЗ-44
   - ФЗ-223
   - ФЗ-275

## Примечания

- Скрипт использует первую найденную компанию и первого пользователя
- Все тендеры создаются с системным шаблоном этапов
- Даты дедлайнов установлены относительно текущей даты
- Для каждого тендера создаётся запись в истории этапов

## Удаление тестовых данных

Если нужно удалить добавленные тендеры:

```sql
DELETE FROM tenders 
WHERE purchase_number IN (
  '0373200000123000010',
  '0373300000123000020',
  '0373400000123000030'
);
```
